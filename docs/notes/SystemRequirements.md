> [!Question] Проблема 
> У наших клиентов (пользоваталей, обычных работяг) скопилось множество фотографий счетчиков. Клиенты хотят получить статистику по каждому счетчику, не переписывая вручную все в какую-то эксель таблицу. А им хотелось бы загрузить уже имеющиеся фотографии, и чтобы на выходе автоматически рисовались все интересующие их графики.

> [!Abstract] Требования 
> > [!todo] От меня
> > Стратегия **MVP**
> > - Время обработки одного изображения - до 15 секунд
> > - Поддержка форматов .jpeg, .png, 
> > - Максимальный размер изображения < 13 МБ
> > - Максимальное разрешение 8192x6144  
> > - Подведение статистики данных должно проводиться локально (Это не проблема нашего ML pipeline)
> > - REST API для ML сервера
> > - Docker для ML сервера
> > 
> > Стратегия **MVP+**
> > - GPU free версия ML
> > - Асинхронное обслуживание 20+ человек одновременно (с GPU)
> > - Кеширование результатов ML сервиса//Конечных исправлений для фотографии hash(Users_Image)
> > - Реализация других способов получения даты фотографии, вне зависимости от наличия соответствующих правильных метаданных (например если фотография была скачана как фото через телеграм)
> > 
> > Стратегия **MVP++**
> > - Serverless ML - то есть прямо на устройстве пользователя
> > - Добавление фотографии напрямую через фото с камеры устройства
> > - Экспорт статистики
> > 	- В какой-то свой формат / или в какой-то .csv, .xlxs
> > - Импорт статистики
> > 	- Из нашего формата
> > - Отправление уведомлений раз в месяц
> > - Автоматическое предложение фотографий счетчиков при выборе
> > - Автоматическое распределение фотографий по "папкам" исходя из метаданных фотографии (место, время, с какого устройства снимали)
